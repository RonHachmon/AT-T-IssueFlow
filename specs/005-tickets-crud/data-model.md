# Data Model: Tickets

## Entity: Ticket

The `Ticket` entity is the core work item in IssueFlow. It belongs to exactly one `Project` and may optionally be assigned to one `User`.

### Fields

| Field        | Java Type       | Column          | Nullable | Notes                                                                 |
|--------------|-----------------|-----------------|----------|-----------------------------------------------------------------------|
| `id`         | `Long`          | `id`            | NO       | BIGSERIAL primary key, generated by DB                                |
| `title`      | `String`        | `title`         | NO       | Max 255 chars; validated with `@NotBlank @Size(max=255)`             |
| `description`| `String`        | `description`   | YES      | Free-form TEXT; no length constraint                                  |
| `status`     | `TicketStatus`  | `status`        | NO       | `@Enumerated(STRING)`; DEFAULT 'TODO'; values: TODO, IN_PROGRESS, IN_REVIEW, DONE |
| `priority`   | `TicketPriority`| `priority`      | NO       | `@Enumerated(STRING)`; values: LOW, MEDIUM, HIGH, CRITICAL           |
| `type`       | `TicketType`    | `type`          | NO       | `@Enumerated(STRING)`; values: BUG, FEATURE, TECHNICAL               |
| `project`    | `Project`       | `project_id`    | NO       | `@ManyToOne(LAZY)`, FK → `projects(id)`, not updatable via PATCH     |
| `assignee`   | `User`          | `assignee_id`   | YES      | `@ManyToOne(LAZY)`, FK → `users(id)`, nullable                       |
| `dueDate`    | `Instant`       | `due_date`      | YES      | Optional deadline; used to compute `isOverdue`                       |
| `createdAt`  | `Instant`       | `created_at`    | NO       | Set by `@PrePersist`; not updatable                                   |
| `updatedAt`  | `Instant`       | `updated_at`    | NO       | Set by `@PrePersist`; refreshed by `@PreUpdate`                      |
| `deletedAt`  | `Instant`       | `deleted_at`    | YES      | `null` = active; non-null = soft-deleted. Set explicitly by service. |
| `version`    | `Long`          | `version`       | NO       | JPA `@Version` for optimistic locking; DB DEFAULT 0; never exposed in responses |

### Enums

```
TicketStatus  : TODO(0), IN_PROGRESS(1), IN_REVIEW(2), DONE(3)
TicketPriority: LOW, MEDIUM, HIGH, CRITICAL
TicketType    : BUG, FEATURE, TECHNICAL
```

Ordinal values are used internally to enforce forward-only transitions: the target status ordinal must equal the current status ordinal + 1.

### Relationships

```
Ticket  →  Project  (ManyToOne, NOT NULL)
Ticket  →  User     (ManyToOne, nullable — the assignee)
```

No changes to `Project` or `User` entities in this phase.

### State Machine

```
TODO ──► IN_PROGRESS ──► IN_REVIEW ──► DONE
  ^            ^               ^
  │            │               │
  └────────────┴───────────────┘
         (backward transitions REJECTED)
```

- Transitions: only `current.ordinal() + 1` is accepted
- DONE tickets: no field updates and no transitions
- Soft-delete is independent of status (a DONE ticket can still be soft-deleted)

## Flyway Migration: V6__tickets.sql

```sql
CREATE TABLE tickets (
    id           BIGSERIAL       PRIMARY KEY,
    title        VARCHAR(255)    NOT NULL,
    description  TEXT,
    status       VARCHAR(20)     NOT NULL DEFAULT 'TODO',
    priority     VARCHAR(10)     NOT NULL,
    type         VARCHAR(15)     NOT NULL,
    project_id   BIGINT          NOT NULL REFERENCES projects(id),
    assignee_id  BIGINT          REFERENCES users(id),
    due_date     TIMESTAMPTZ,
    created_at   TIMESTAMPTZ     NOT NULL,
    updated_at   TIMESTAMPTZ     NOT NULL,
    deleted_at   TIMESTAMPTZ,
    version      BIGINT          NOT NULL DEFAULT 0
);

-- Efficient lookups: active tickets per project ordered by id
CREATE INDEX ix_tickets_project_active
    ON tickets (project_id, id)
    WHERE deleted_at IS NULL;
```

## New Error Class: InvalidStateTransitionException

```java
// src/main/java/com/att/tdp/issueflow/common/error/InvalidStateTransitionException.java
public class InvalidStateTransitionException extends RuntimeException {
    public InvalidStateTransitionException(String reason) {
        super(reason);
    }
}
```

Maps to `409 Conflict` via a new handler in `GlobalExceptionHandler` using `ProblemDetailFactory.conflict(exception.getMessage())`.

## DTOs

### CreateTicketRequest

| Field         | Type            | Constraints                         |
|---------------|-----------------|-------------------------------------|
| `title`       | `String`        | `@NotBlank`, `@Size(max=255)`       |
| `description` | `String`        | optional                            |
| `priority`    | `TicketPriority`| `@NotNull`                          |
| `type`        | `TicketType`    | `@NotNull`                          |
| `projectId`   | `Long`          | `@NotNull`                          |
| `assigneeId`  | `Long`          | optional                            |
| `dueDate`     | `Instant`       | optional                            |

Note: `status` is NOT in the create request — it defaults to `TODO`.

### UpdateTicketRequest

At least one field must be non-null (validated via a class-level `@AssertTrue` or service guard).

| Field         | Type            | Constraints |
|---------------|-----------------|-------------|
| `title`       | `String`        | optional; `@Size(max=255)` if present |
| `description` | `String`        | optional    |
| `priority`    | `TicketPriority`| optional    |
| `type`        | `TicketType`    | optional    |
| `assigneeId`  | `Long`          | optional; `null` means "clear assignee" (use `Optional<Long>`) |
| `dueDate`     | `Instant`       | optional    |
| `status`      | `TicketStatus`  | optional; forward-only transitions enforced in service |

### TicketResponse

| Field         | Type            | Notes                                                  |
|---------------|-----------------|--------------------------------------------------------|
| `id`          | `Long`          |                                                        |
| `title`       | `String`        |                                                        |
| `description` | `String`        | nullable                                               |
| `status`      | `TicketStatus`  |                                                        |
| `priority`    | `TicketPriority`|                                                        |
| `type`        | `TicketType`    |                                                        |
| `projectId`   | `Long`          | mapped from `ticket.project.id`                        |
| `assigneeId`  | `Long`          | nullable; mapped from `ticket.assignee.id`             |
| `dueDate`     | `Instant`       | nullable                                               |
| `isOverdue`   | `boolean`       | computed: `dueDate != null && dueDate.isBefore(now())` |
